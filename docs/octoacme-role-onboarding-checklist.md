# OctoAcme Role Onboarding Checklist

This supplementary guide helps new team members ramp up quickly by providing role-specific orientation tasks, key artifacts to review, and first-week collaboration expectations.

---

## How to Use This Checklist

1. Identify your role in the table of contents below.
2. Complete all orientation tasks in your section during your first week.
3. Review the **Quick-Reference Guide** at the bottom to familiarize yourself with the responsibilities of other roles.

## Developer

### Orientation Tasks
- [ ] Set up local development environment following the repository README
- [ ] Complete access provisioning (GitHub, CI/CD, cloud console, monitoring tools)
- [ ] Attend your first standup and introduce yourself to the team
- [ ] Review the team's branching strategy and pull request conventions
- [ ] Pair with an existing Developer on a starter issue or good-first-issue ticket

### Key Artifacts to Review
- [ ] `docs/octoacme-roles-and-personas.md` — understand all team roles
- [ ] `docs/octoacme-project-management-overview.md` — project lifecycle overview
- [ ] `docs/octoacme-execution-and-tracking.md` — tracking and delivery practices
- [ ] `docs/octoacme-release-and-deployment.md` — release and deployment process
- [ ] Current sprint board and backlog in the project tracker

### First-Week Collaboration Expectations
- Shadow a sprint planning session with the **Agile Coach / Scrum Master**
- Sync with the **Project Manager** on current team commitments and milestones
- Review any open design specs with the **UX Designer**
- Understand the CI/CD pipeline with the **DevOps Engineer**

### Key Artifacts to Review

### First-Week Collaboration Expectations

- Meet with the **Project Manager** to understand delivery commitments and constraints

## DevOps Engineer

### Orientation Tasks
- [ ] Review existing CI/CD pipeline architecture and deployment runbooks
- [ ] Complete access provisioning (GitHub, cloud console, monitoring tools, secrets management)
- [ ] Conduct an infrastructure walkthrough with an existing DevOps Engineer or senior Developer
- [ ] Review current monitoring dashboards and alerting rules
- [ ] Understand the release and deployment process end-to-end

### Key Artifacts to Review
- [ ] `docs/octoacme-roles-and-personas.md` — understand all team roles
- [ ] `docs/octoacme-release-and-deployment.md` — release and deployment process
- [ ] `docs/octoacme-execution-and-tracking.md` — how development work flows to release
- [ ] CI/CD pipeline configuration files and infrastructure-as-code repositories
- [ ] Incident post-mortem reports and on-call runbooks

### First-Week Collaboration Expectations
- Walk through the deployment pipeline with **Developers**
- Align with the **Project Manager** on upcoming release windows
- Review infrastructure documentation coverage with the **Technical Writer**
- Flag any current infrastructure risks with the **Agile Coach / Scrum Master**

## Technical Writer

### Orientation Tasks
- [ ] Audit the existing documentation in the `docs/` folder for coverage and quality
- [ ] Complete access provisioning (GitHub, documentation tools, project tracker)
- [ ] Meet with each role lead to understand documentation pain points
- [ ] Review the documentation style guide (or create one if it does not exist)
- [ ] Identify the highest-priority documentation gaps and create tracking issues

### Key Artifacts to Review
- [ ] All files in `docs/` — audit for completeness and accuracy
- [ ] `docs/octoacme-roles-and-personas.md` — understand all team roles
- [ ] `docs/octoacme-project-management-overview.md` — overall process framework
- [ ] Recent sprint review notes for undocumented feature changes
- [ ] Existing onboarding materials and README files

### First-Week Collaboration Expectations
- Sync with the **Project Manager** on documentation milestones and release notes needs
- Meet with **Developers** to understand undocumented technical processes
- Align with the **UX Designer** on in-product copy and user-facing documentation
- Review infrastructure runbook gaps with the **DevOps Engineer**

## Quick-Reference Guide to Other Roles

Use this table to quickly understand what each role does and when to reach out to them.

| Role                        | Primary Focus                                      | When to Contact                                                     |
|-----------------------------|----------------------------------------------------|---------------------------------------------------------------------|
| **Developer**               | Building and testing features                      | Technical implementation questions, PR reviews, bug investigation   |
| **Product Manager**         | Product vision, backlog, and success metrics       | Prioritisation decisions, feature requirements, user feedback       |
| **Project Manager**         | Schedule, risks, and cross-team coordination       | Milestone tracking, dependency management, stakeholder updates      |
| **UX Designer**             | User research, wireframes, and design systems      | Design specs, usability feedback, in-product copy                   |
| **Agile Coach / Scrum Master** | Agile ceremonies and continuous improvement     | Blockers, process questions, retrospective facilitation             |
| **DevOps Engineer**         | CI/CD, infrastructure, and deployment              | Pipeline issues, release support, environment access                |
| **Technical Writer**        | Documentation and knowledge management             | Process documentation, runbooks, onboarding guides, release notes   |
